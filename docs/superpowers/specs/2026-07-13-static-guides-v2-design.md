# 静态教程 V2 大型重构设计规格

**日期：** 2026-07-13

**状态：** 已完成

**适用仓库：** SubAPI

**视觉方向：** A「云端蓝图」

## 1. 背景

当前静态教程由 1 个父教程和 6 个客户端子教程组成。网站端使用 Vue 组件维护正文，Markdown 用于 Word/飞书导出，二者依赖人工同步，已经出现内容漂移。现有页面还存在双 H1、移动端先展示长目录、首屏入口重复、公共排错内容分散、素材命名不可维护、旧截图过时或包含敏感信息等问题。

本次在完整保留 V1 的前提下建设 V2。V2 不覆盖或重定向 V1，而是使用独立路由、组件、样式、内容与素材目录。网站、Markdown 和 Word/飞书成品由同一份 V2 Markdown 源稿驱动。

## 2. 已确认决策

| 项目 | 决策 |
| --- | --- |
| 页面范围 | 教程中心、快速开始、6 个客户端教程、公共排错中心，共 9 页 |
| V1 兼容 | 7 个原入口、路由名、`guideKey`、组件绑定与公开访问行为保持不变 |
| 内容同步 | 网站 V2、Markdown、Word/飞书同步重构 |
| 内容架构 | 独立 V2 + Markdown 单一内容源 |
| 视觉方向 | A「云端蓝图」：明亮蓝灰、克制留白、清晰线性路径 |
| 品牌策略 | 网站读取部署环境公开站点名称；导出成品使用“AI 客户端使用指南” |
| 进度状态 | 仅保存在浏览器本地，不上传服务器，可一键清除 |
| 媒体策略 | 图片为主，复杂连续操作才使用 45–90 秒短视频 |

## 3. 目标与非目标

### 3.1 目标

1. 新手无需预先理解 API，即可在 5–10 分钟内找到目标客户端并完成配置。
2. 首页负责“选路”，详情页一次只推进一个任务，减少首屏信息过载。
3. 网站、Markdown 和 Word/飞书正文只维护一份，消除人工同步漂移。
4. V2 页面、CSS 与媒体完全隔离，避免误伤 V1。
5. 所有关键步骤具备完成标志、文字替代和可恢复错误状态。
6. 390px、768px、1440px 三档均具备专业、清晰、触控友好的体验。
7. 素材不包含真实 API Key、账号名、余额、用量或已过时域名。

### 3.2 非目标

1. 不修改、删除或重定向 V1 页面和 V1 依赖的旧素材。
2. 不为教程新增后端进度接口、账号同步或跨设备同步。
3. 不引入通用 CMS、数据库内容管理或在线富文本编辑器。
4. 不把视频作为完成教程的唯一方式。
5. 不在本次顺带重构站内其他公共页面或全局设计系统。

## 4. 路由与信息架构

### 4.1 V1 冻结入口

- `/registration-key-guide`
- `/codex-guide`
- `/claude-code-guide`
- `/open-code-guide`
- `/open-claw-guide`
- `/mobile-guide`
- `/image-guide`

上述路径不重定向到 V2，也不改为加载 V2 组件。

### 4.2 V2 新入口

| 页面 | 路径 | 职责 |
| --- | --- | --- |
| 教程中心 | `/guides/v2` | 解释完整流程、选择客户端、进入快速开始或直接配置 |
| 快速开始 | `/guides/v2/get-started` | 注册、获取权益、创建 API Key、选择客户端 |
| Codex | `/guides/v2/codex` | Codex 下载、初始化、配置、登录与验证 |
| Claude Code | `/guides/v2/claude-code` | 安装、配置方式选择与验证 |
| OpenCode | `/guides/v2/opencode` | 安装、配置文件、临时连接与验证 |
| OpenClaw | `/guides/v2/openclaw` | 先选云端或本地，再进入对应步骤 |
| Chatbox 移动端 | `/guides/v2/chatbox-mobile` | iOS/Android 安装、提供方、模型与测试 |
| Cherry Studio 生图 | `/guides/v2/cherry-studio-image` | 模型服务、图像模型、生成与验证 |
| 排错中心 | `/guides/v2/troubleshooting` | 401、404、429、模型不可用、配置未生效与联系支持 |

所有 V2 路由均为匿名公开页面，并在 Backend Mode 的公开访问逻辑中显式放行。无效 V2 slug 显示专用教程 404，引导返回教程中心。

## 5. 用户体验与视觉系统

### 5.1 教程中心

首页首屏只完成三件事：

1. 用“账户 → Key → 客户端 → 测试”解释完整路径。
2. 让用户选择目标客户端。
3. 提供“从零开始”和“我已有 API Key”两个入口。

客户端卡片展示平台、预计耗时、难度和主要配置方式，不在首屏堆叠配置字段。页面提供旧版教程入口，但不修改 V1 页面反向引流。

### 5.2 教程详情页

详情页保持单一 H1，并依次展示：

1. 面包屑、标题、平台、预计耗时和最后核验日期。
2. 开始前检查和完成本教程后可得到的结果。
3. 当前步骤、完成标志和下一步操作。
4. 平台分支，例如 Windows、macOS、Linux。
5. 专属排错与公共排错中心入口。

桌面端使用轻量粘性步骤导航；移动端先显示标题和开始按钮，目录放入标题后的折叠抽屉。不会再让移动用户先滚过长目录。

### 5.3 云端蓝图视觉

- 主色为克制的蓝色，搭配蓝灰文字、白色内容面和低对比边框。
- 使用大留白、清晰字号层级、少量柔和渐变建立高级感。
- 避免容器套容器；步骤块只在需要分组或强调完成结果时使用边界。
- 动画只用于状态反馈和轻量进入效果，并遵循 `prefers-reduced-motion`。
- 移动端触控目标保持约 44px，避免横向滚动、固定侧栏遮挡和超长按钮文字。

### 5.4 本地进度

步骤完成状态、上次阅读位置和平台偏好仅保存到浏览器本地存储：

- 不要求登录。
- 不上传服务器。
- 存储损坏或浏览器禁用存储时退化为无持久化模式。
- 用户可在页面内清除本教程或全部 V2 进度。
- Word/飞书成品展开全部平台分支，不依赖任何交互状态。

## 6. 单一内容源

### 6.1 内容位置

V2 源稿放在前端可构建目录中，避免 Docker 构建额外复制仓库外内容：

```text
frontend/src/content/guides-v2/
├── index.md
├── get-started.md
├── codex.md
├── claude-code.md
├── opencode.md
├── openclaw.md
├── chatbox-mobile.md
├── cherry-studio-image.md
└── troubleshooting.md
```

每份文件由 YAML Frontmatter 和标准 Markdown 正文组成。正文不写 Vue 私有组件标签，保证源稿可直接阅读、测试和导出。

### 6.2 Frontmatter

每篇至少包含：

```yaml
title: 配置 Codex
slug: codex
summary: 使用 API Key 完成 Codex 配置与连接验证
duration: 8 分钟
platforms:
  - Windows
  - macOS
  - Linux
difficulty: 新手
updatedAt: 2026-07-13
version: v2
```

易变化的模型名、API 类型、分组名、域名和支持入口集中在明确的数据区或共享配置中，页面展示“最后核验日期”，避免把时效信息散落在多篇正文。

### 6.3 共享解析层

共享解析层把 Markdown 规范化为 `GuideDocument`：

- 校验必填元数据、slug 唯一性和允许的平台值。
- 生成目录、步骤编号、上一篇/下一篇关系和页面标题。
- 校验内部链接、外部链接协议和媒体路径。
- 过滤不安全 HTML，禁止脚本、事件属性和危险 URL。
- 为网站渲染器和导出脚本提供一致的章节语义。

构建时遇到元数据、链接或媒体缺失应直接失败，不静默发布残缺页面。

## 7. 前端组件与文件边界

建议结构：

```text
frontend/src/views/public/guides-v2/
├── GuideV2HubView.vue
├── GuideV2View.vue
├── GuideV2NotFoundView.vue
├── guide-v2.registry.ts
├── guide-v2.types.ts
├── guide-v2.parser.ts
├── components/
│   ├── GuideV2Header.vue
│   ├── GuideV2Hero.vue
│   ├── GuideV2Sidebar.vue
│   ├── GuideV2MobileToc.vue
│   ├── GuideV2Renderer.vue
│   ├── GuideV2Step.vue
│   ├── GuideV2Media.vue
│   ├── GuideV2CodeBlock.vue
│   ├── GuideV2Notice.vue
│   └── GuideV2Support.vue
├── composables/
│   └── useGuideV2Progress.ts
└── styles/
    ├── tokens.css
    ├── layout.css
    ├── content.css
    └── responsive.css
```

职责边界：

- `registry` 负责路径、路由名、源稿加载器和前后篇关系。
- `parser` 负责输入校验、Markdown token、安全过滤和 `GuideDocument` 输出。
- `renderer` 只负责把规范化 token 映射为 V2 展示组件。
- `progress` 只负责本地状态，不依赖认证 store 或后端 API。
- `styles` 全部使用 `.guide-v2-*` 命名空间，不复用或覆盖 V1 的 `.codex-*` 选择器。
- 每篇 Markdown 按路由懒加载，进入一篇教程时不下载其他正文。

## 8. 图片与视频

### 8.1 目录

```text
frontend/public/img/guides/v2/
├── common/
├── codex/
├── claude-code/
├── opencode/
├── openclaw/
├── chatbox/
└── cherry-studio/
```

素材使用语义化名称，例如 `create-key-select-group.webp`、`codex-windows-config-folder.webp`。V2 不复用含真实信息、旧域名或低质量手绘标注的 V1 截图。

### 8.2 图片规范

- 优先提供 WebP；关键大图可补充 AVIF 或高分辨率版本。
- 设置明确 `width`、`height`、`srcset` 和 `sizes`，减少布局跳动。
- 首屏关键媒体不懒加载，正文媒体使用原生懒加载。
- 使用统一蓝色编号、高亮框、箭头、圆角和说明文字。
- 所有画面使用模拟账号、示例余额和不可用 Key。
- 图片必须有可理解的 `alt` 和相邻文字步骤；移动端支持点开放大。

### 8.3 视频规范

- 仅用于连续操作明显比多张图片更易理解的流程。
- 目标时长 45–90 秒。
- 提供封面、字幕、播放器控件和静音可理解的画面标注。
- 不自动播放、不无限循环，默认 `preload="metadata"`。
- 视频失败时仍保留完整文字步骤。

### 8.4 素材清单

维护 V2 素材清单，至少记录：路径、用途、来源、客户端版本、更新时间、是否模拟数据、对应教程与步骤。提交前扫描疑似 API Key、旧域名和未引用文件。

## 9. 错误处理与可访问性

### 9.1 页面错误

- 无效 V2 路径：显示教程专用 404 和返回教程中心按钮。
- Markdown 解析失败：开发环境显示明确文件和字段，生产构建失败。
- 图片或视频加载失败：展示说明、重试入口和文字步骤，不留空白占位。
- 复制失败：提示手动选择文本，不静默吞错。
- 本地存储失败：降级为当前会话状态，页面仍可继续使用。

### 9.2 内容错误

公共的 401、404、429、模型不可用和配置未生效统一进入排错中心。客户端页只保留该客户端特有问题，并提供准确的公共排错入口。

### 9.3 可访问性

- 每页只有一个 H1，标题层级顺序正确。
- 目录、平台切换、复制按钮、进度控件可使用键盘操作。
- 使用可见焦点、足够颜色对比和非颜色状态提示。
- 动画遵循 reduced-motion。
- 图片、视频和交互步骤都有文字等价内容。

## 10. Word/飞书导出

现有 Word 导出流程扩展为读取 `frontend/src/content/guides-v2/`。输出 9 份 V2 Word 成品，并保留 V1 导出能力。

导出规则：

- 导出品牌统一为“AI 客户端使用指南”。
- 保留标题层级、列表、代码块、提示、表格和图片说明。
- 图片直接内嵌到 `.docx`。
- 展开全部平台分支和完整文字步骤。
- 视频转换为封面图、说明和可点击链接，不让导出文档依赖播放器。
- `--check` 校验成品与 V2 Markdown 源稿一致。

## 11. 测试与验收

### 11.1 TDD 顺序

所有行为变更遵循 RED → GREEN → REFACTOR：先编写失败测试并确认失败原因，再做最小实现，随后重构并保持测试通过。

### 11.2 自动化测试

1. **V1 冻结测试**
   - 精确断言 7 个旧路径、路由名、`guideKey` 和组件绑定。
   - 匿名与 Backend Mode 公开访问行为不变。

2. **内容与安全测试**
   - Frontmatter 必填字段、slug 唯一、允许值和更新时间格式。
   - 内部链接、外部协议、媒体文件存在性。
   - 危险 HTML、事件属性和危险 URL 被拒绝或清理。
   - 源稿与媒体不包含疑似真实 API Key、旧域名和禁止示例。

3. **V2 路由测试**
   - 9 个路径、标题、公开访问和深层刷新。
   - 无效 slug 进入 V2 404。
   - hash 章节定位考虑固定页眉偏移。

4. **组件测试**
   - 单一 H1、目录、平台切换、上一篇/下一篇。
   - 本地进度保存、损坏恢复和清除。
   - 复制成功/失败、媒体成功/失败和文字替代。
   - 键盘交互和 reduced-motion。

5. **导出测试**
   - 9 份 Word 文件生成。
   - 标题层级、代码块、表格、图片内嵌和平台分支完整。
   - `--check` 能发现源稿与成品漂移。

6. **工程验证**
   - 相关 Vitest。
   - 覆盖率检查，新增核心模块达到仓库要求。
   - ESLint、TypeScript、Vite build。
   - Go 内嵌前端的最小构建或对应后端验证。

### 11.3 浏览器验收

使用可用的 Browser 工具优先验证，至少覆盖 1440×900、768×1024、390×844：

1. 打开 `/guides/v2`，确认动态站点品牌、首屏层次和客户端入口。
2. 选择 Codex；分别验证“从零开始”和“已有 API Key”路径。
3. 切换 Windows、macOS、Linux，刷新后偏好仍保留。
4. 完成步骤、复制配置、模拟媒体失败，确认可见反馈与恢复入口。
5. 进入排错中心并返回教程；移动端无横向溢出、目录遮挡或触控困难。
6. 检查页面身份、非空内容、无框架错误层、控制台无相关错误，并保存桌面和移动端截图证据。

## 12. 完成标准

只有同时满足以下条件才可标记完成：

- V1 7 个入口与行为全部通过冻结测试。
- V2 9 个页面可公开访问并完成关键流程。
- 网站、Markdown 和 Word/飞书来自同一内容源，导出检查无漂移。
- 390px、768px、1440px 浏览器验收通过。
- 新素材无真实密钥、账号、余额、用量或旧域名。
- 相关测试、类型检查、静态检查和构建通过。
- 无未说明的重大视觉偏差、控制台错误或缺失媒体。
- 提交仅包含本任务文件，不带入现有未跟踪运维文件或其他并行改动。

## 13. 风险与控制

| 风险 | 控制措施 |
| --- | --- |
| V2 路由误伤 V1 或 Backend Mode | 精确冻结 7 个 V1 路由；V2 使用独立前缀与公开访问测试 |
| Markdown 与富交互能力冲突 | 标准 Markdown 保留核心内容；Vue 只做渐进增强，不隐藏关键步骤 |
| 素材过时或泄露真实数据 | 模拟数据、素材清单、敏感信息扫描、最后核验日期 |
| CSS 串色 | 独立目录和 `.guide-v2-*` 命名空间，不导入 V1 样式 |
| Word/飞书无法表达交互 | 展开平台分支；视频输出封面、说明和链接 |
| 本地进度状态损坏 | 输入校验、版本化存储和无持久化降级 |
| 大量媒体拖慢移动端 | WebP/AVIF、固定尺寸、响应式资源、首屏加载策略 |
| 并行工作区改动冲突 | 实施前重新检查工作区，只暂存本任务文件，重叠文件先核对差异 |

## 14. 实施前置

本规格复核通过后，使用 writing-plans 技能编写逐任务实施计划。计划应按 TDD 拆分，明确每一步的失败测试、最小实现、验证命令和提交边界；随后再开始任何生产代码修改。
